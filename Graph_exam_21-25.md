### 21. Расстояние редактирования графа (Graph Edit Distance, GED)

**GED** — это метрика, которая позволяет оценить, насколько два графа похожи друг на друга. Она аналогична расстоянию Левенштейна для строк. Суть в том, чтобы найти минимальную стоимость преобразования графа $G_1$ в граф $G_2$.

**Операции редактирования:**
Каждое изменение в графе имеет определенную "стоимость" (cost), заданную функцией весов:
1.  Вставка вершины (Node insertion).
2.  Удаление вершины (Node deletion).
3.  Изменение метки/атрибута вершины (Node substitution).
4.  Вставка ребра (Edge insertion).
5.  Удаление ребра (Edge deletion).
6.  Изменение метки/веса ребра (Edge substitution).

**Формальное определение:**
GED между $G_1$ и $G_2$ — это минимум суммы стоимостей операций по всем возможным последовательностям операций, переводящим $G_1$ в $G_2$.

**Особенности:**
*   **Сложность:** Вычисление точного GED — **NP-трудная задача**, так как она сводится к задаче о поиске изоморфного подграфа. Для больших графов используют эвристики и аппроксимации.
*   **Применение:** Распознавание образов, компьютерное зрение, хемоинформатика (сравнение молекулярных структур, представленных в виде графов), анализ социальных сетей.

### 22. Алгоритмы Прима и Краскала

Оба алгоритма решают задачу поиска **Минимального остовного дерева (Minimum Spanning Tree, MST)** в связном неориентированном взвешенном графе. Суть: нужно выбрать подмножество ребер, которые связывают все вершины без образования циклов, причем сумма весов этих ребер должна быть минимальной.

**А. Алгоритм Краскала (Kruskal's algorithm)**
*   **Идея:** "Жадный" подход со стороны *ребер*. 
    1. Сортируем все ребра графа по возрастанию их веса.
    2. Изначально каждая вершина — это отдельное дерево (лес).
    3. Идем по отсортированному списку ребер. Если ребро соединяет вершины из *разных* деревьев (то есть не образует цикла), мы добавляем его в MST и объединяем эти два дерева.
*   **Структура данных:** Для быстрой проверки на циклы и объединения деревьев используется **СНМ (Система Непересекающихся Множеств / Disjoint Set Union, DSU)**.
*   **Сложность:** $O(E \log E)$ или $O(E \log V)$ — определяется временем сортировки ребер.

**Б. Алгоритм Прима (Prim's algorithm)**
*   **Идея:** "Жадный" подход со стороны *вершин* (похож на алгоритм Дейкстры).
    1. Начинаем с любой произвольной вершины, помещая её в растущее MST.
    2. На каждом шаге смотрим на все ребра, которые соединяют уже добавленные в MST вершины с *ещё не добавленными*.
    3. Выбираем ребро с **минимальным весом** и добавляем его и новую вершину в MST.
    4. Повторяем, пока все $V$ вершин не окажутся в дереве.
*   **Структура данных:** Очередь с приоритетом (Min-Heap) для быстрого извлечения минимального ребра.
*   **Сложность:** 
    * С использованием бинарной кучи (для разреженных графов): $O(E \log V)$.
    * С обычным массивом (для плотных графов): $O(V^2)$.

### 23. Алгоритм Форда — Беллмана (Bellman-Ford)

Решает задачу поиска кратчайших путей от **одной стартовой вершины** до всех остальных во взвешенном ориентированном графе.
*   **Главное отличие:** В отличие от Дейкстры, алгоритм Форда-Беллмана корректно работает с графами, где есть **отрицательные веса ребер**.
*   **Обнаружение циклов:** Алгоритм способен определить наличие *цикла отрицательного веса* (если такой цикл достижим из стартовой вершины, то кратчайшего пути не существует, так как можно бесконечно "накручивать" отрицательный вес).

**Суть алгоритма (Динамическое программирование):**
Основан на операции **релаксации (ослабления) ребра**: если расстояние до вершины $v$ можно улучшить, пройдя через вершину $u$ по ребру $(u, v)$ с весом $w$, то мы обновляем его: `if (d[v] > d[u] + w) d[v] = d[u] + w`.

1. Изначально расстояния до всех вершин равны $\infty$, а до стартовой — $0$.
2. Выполняем релаксацию **всех** ребер графа $V - 1$ раз. (Так как любой простой кратчайший путь состоит максимум из $V - 1$ ребра).
3. *Проверка на отрицательный цикл:* Если после $V-1$ итераций мы сделаем еще один (сотый/тысячный) проход по всем ребрам, и какое-то расстояние снова уменьшится, значит, в графе есть отрицательный цикл!

**Сложность:** $O(V \cdot E)$.

### 24. Алгоритм Дейкстры (Dijkstra)

Самый быстрый и популярный алгоритм для поиска кратчайших путей от **одной вершины** до всех остальных.
*   **Важное ограничение:** Алгоритм работает **ТОЛЬКО с неотрицательными весами ребер** ($w \ge 0$). Если есть отрицательные ребра, алгоритм может выдать неверный результат.

**Суть алгоритма ("Жадный"):**
Граф делится на посещенные (расстояние до которых уже окончательно найдено) и непосещенные вершины.
1. Расстояние до старта $0$, до остальных $\infty$.
2. На каждом шаге из всех **непосещенных** вершин выбирается вершина $u$ с **минимальным текущим расстоянием** `d[u]`.
3. Эта вершина $u$ помечается как посещенная (её кратчайший путь найден окончательно).
4. Делаем **релаксацию** всех её соседей: если путь в соседа $v$ через $u$ короче, чем текущий `d[v]`, обновляем `d[v] = d[u] + weight(u, v)`.
5. Повторяем, пока не посетим все доступные вершины.

**Сложность зависит от структуры данных:**
1.  **Массив** (поиск минимума за $O(V)$): Сложность **$O(V^2)$**. Оптимально для *плотных* графов (где ребер много, почти $V^2$).
2.  **Очередь с приоритетом / куча** (поиск минимума за $O(\log V)$): Сложность **$O(E \log V)$**. Оптимально для *разреженных* графов (где ребер мало).

### 25. Алгоритм Флойда (-Уоршелла) (Floyd-Warshall)

Решает задачу поиска кратчайших путей **между ВСЕМИ парами вершин** графа за один запуск.
*   **Особенности:** Граф может быть ориентированным. Работает с **отрицательными весами**.
*   **Поиск отрицательных циклов:** Если после отработки алгоритма на главной диагонали матрицы расстояний (`d[i][i]`) появилось отрицательное число, значит в графе есть отрицательный цикл.

**Суть алгоритма (Динамическое программирование):**
Пусть `d[i][j]` — это кратчайший путь из $i$ в $j$. Идея заключается в постепенном добавлении вершин, которые разрешено использовать в качестве промежуточных (транзитных) на пути.
Мы перебираем вершину $k$ и проверяем: не станет ли путь из $i$ в $j$ короче, если мы пройдем его *через* вершину $k$?

**Код алгоритма (всего 3 цикла):**
```cpp
// Изначально d[i][j] = вес ребра (i,j) или бесконечность, d[i][i] = 0.
for (int k = 1; k <= V; k++) {          // k - промежуточная вершина (!!! ВАЖНО: цикл по k должен быть самым внешним)
    for (int i = 1; i <= V; i++) {      // i - стартовая вершина
        for (int j = 1; j <= V; j++) {  // j - конечная вершина
            if (d[i][k] < INF && d[k][j] < INF) {
                d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
            }
        }
    }
}
```

**Сложность:**
*   **По времени:** $O(V^3)$.
*   **По памяти:** $O(V^2)$ (для хранения матрицы расстояний).
*   Из-за того, что внутри циклов происходят простейшие операции с минимальным ветвлением, алгоритм работает очень быстро на практике (с малыми константами), но применим только для графов до ~500 вершин. Для разреженных графов часто выгоднее $V$ раз запустить алгоритм Дейкстры.
