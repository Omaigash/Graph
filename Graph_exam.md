### 1. Простейшие определения и свойства графов

**Граф** $G = (V, E)$ — это математическая модель, состоящая из множества вершин $V$ (узлов) и множества ребер $E$, соединяющих пары вершин.

**Основные определения:**
*   **Ориентированный граф (орграф):** Ребра имеют направление (называются *дугами*). Пары вершин упорядочены: $(u, v)$.
*   **Неориентированный граф:** Ребра не имеют направления, связь двусторонняя.
*   **Смежность:** Две вершины смежны, если они соединены ребром. Два ребра смежны, если имеют общую вершину.
*   **Инцидентность:** Связь между вершиной и ребром. Вершина инцидентна ребру, если ребро из неё выходит или в неё входит.
*   **Степень вершины $\deg(v)$:** Количество ребер, инцидентных вершине $v$. 
    *   В орграфах выделяют полустепень захода $\deg^+(v)$ и полустепень исхода $\deg^-(v)$.
    *   *Изолированная вершина* — степень 0.
    *   *Висячая вершина (лист)* — степень 1.
*   **Путь:** Последовательность вершин, в которой каждые две соседние соединены ребром. *Простой путь* не содержит повторяющихся вершин.
*   **Цикл:** Путь, у которого начальная и конечная вершины совпадают.
*   **Связный граф:** Граф, в котором между любыми двумя вершинами существует путь.

**Базовые свойства:**
*   **Лемма о рукопожатиях:** Сумма степеней всех вершин любого неориентированного графа равна удвоенному количеству ребер: $\sum \deg(v) = 2|E|$. 
    *   *Следствие:* В любом графе количество вершин нечетной степени всегда четно.
*   **Дерево:** Связный граф без циклов. В дереве всегда $|E| = |V| - 1$.
*   **Полный граф ($K_n$):** Граф, в котором каждые две вершины соединены ребром. Количество ребер равно $\frac{n(n-1)}{2}$.

### 2. Методы представления данных: матрицы инциденций, смежности, списки инцидентности

Для программной обработки графов используются разные структуры данных. Выбор зависит от плотности графа (количества ребер) и решаемой задачи. Пусть $V$ — количество вершин, $E$ — количество ребер.

**А. Матрица смежности (Adjacency Matrix)**
*   **Описание:** Квадратная матрица $A$ размером $|V| \times |V|$. Элемент $A[i][j] = 1$ (или равен весу ребра), если есть ребро из $i$ в $j$, и $0$, если ребра нет.
*   **Свойства:** Для неориентированного графа матрица симметрична относительно главной диагонали.
*   **Сложность по памяти:** $O(V^2)$.
*   **Плюсы:** Проверка наличия ребра между $u$ и $v$ занимает $O(1)$. Удобно для плотных графов.
*   **Минусы:** Требует много памяти. Поиск всех соседей вершины занимает $O(V)$ времени.

**Б. Матрица инцидентности (Incidence Matrix)**
*   **Описание:** Матрица $B$ размером $|V| \times |E|$ (строки — вершины, столбцы — ребра).
*   **Свойства:** 
    *   Для неориентированного графа: $B[i][j] = 1$, если вершина $i$ инцидентна ребру $j$, иначе $0$. В каждом столбце ровно две единицы.
    *   Для орграфа: $1$ (ребро исходит из вершины), $-1$ (ребро входит в вершину), $0$ (не инцидентны).
*   **Сложность по памяти:** $O(V \cdot E)$.
*   **Плюсы:** Удобна для теоретических выкладок и алгебраической топологии.
*   **Минусы:** На практике в программировании используется крайне редко из-за огромного расхода памяти и неэффективности базовых операций.

**В. Списки смежности / инцидентности (Adjacency Lists)**
*   **Описание:** Массив или хеш-таблица размером $|V|$, где каждому элементу (вершине) соответствует список её соседей (или исходящих ребер).
*   **Сложность по памяти:** $O(V + E)$.
*   **Плюсы:** Идеально для разреженных графов (где ребер мало). Перебор всех соседей вершины $v$ занимает время, пропорциональное её степени $O(\deg(v))$.
*   **Минусы:** Проверка наличия конкретного ребра $(u, v)$ в худшем случае занимает $O(\deg(u))$ времени (дольше, чем $O(1)$ у матрицы смежности).

### 3. Стратегии поиска в глубину и в ширину

Это два базовых алгоритма обхода графа. Оба работают за **$O(V + E)$** при использовании списков смежности.

**Поиск в глубину (DFS — Depth-First Search)**
*   **Идея:** Идем "вглубь" графа по ветке до тех пор, пока есть непосещенные смежные вершины. Как только заходим в тупик, возвращаемся на шаг назад (backtracking) и пробуем другой путь.
*   **Структура данных:** Стек (обычно реализуется неявно через рекурсию).
*   **Массив посещений:** Используется массив `visited` (или цвета: белый, серый, черный), чтобы не зациклиться.
*   **Применение:** Топологическая сортировка, поиск компонент связности, проверка на ацикличность (поиск циклов), поиск мостов и шарниров.

**Поиск в ширину (BFS — Breadth-First Search)**
*   **Идея:** Обходим граф "слоями". Сначала посещаем стартовую вершину, затем всех её соседей (на расстоянии 1 ребра), затем соседей соседей (на расстоянии 2) и т.д.
*   **Структура данных:** Очередь (Queue). FIFO (First In, First Out).
*   **Применение:** Поиск кратчайшего пути в *невзвешенном* графе (в смысле наименьшего количества ребер), нахождение расстояний между вершинами.

### 4. Поиск мостов и шарниров (алгоритм Тарьяна и Хопкрофта)

Эти элементы критически важны для оценки надежности сетей.
*   **Мост** — ребро, удаление которого увеличивает число компонент связности графа.
*   **Шарнир (точка сочленения)** — вершина, удаление которой (вместе со всеми инцидентными ей ребрами) увеличивает число компонент связности графа.

**Алгоритм на основе DFS ($O(V + E)$):**
В процессе DFS мы строим так называемое "дерево обхода в глубину". У каждого ребра есть направление обхода. Ребра, не вошедшие в дерево, называются *обратными ребрами* (они ведут к предкам).

Алгоритм поддерживает два массива для каждой вершины $v$:
1.  `tin[v]` (time in) — время (или номер) входа в вершину алгоритмом DFS.
2.  `up[v]` (или `low[v]`) — минимальное время входа `tin`, которого можно достичь из вершины $v$ (или из её потомков в дереве DFS), используя не более одного *обратного ребра*.
    *   Формула пересчета: `up[v] = min(tin[v], up[to])` для ребер дерева, и `up[v] = min(up[v], tin[to])` для обратных ребер (где `to` — сосед `v`).

**Условие моста:**
Ребро $(v, to)$, принадлежащее дереву DFS, является мостом тогда и только тогда, когда:
`up[to] > tin[v]`
*(Это означает, что из поддерева `to` нет ни одного обратного пути, который возвращался бы к `v` или выше неё. Значит, единственный путь туда — само ребро $(v, to)$).*

**Условие шарнира:**
Вершина $v$ является шарниром, если для какого-либо её потомка `to` (в дереве DFS) выполняется:
`up[to] >= tin[v]`
*(Из поддерева потомка `to` нельзя подняться строго "выше" вершины $v$).*
*Исключение:* Корень дерева DFS является шарниром тогда и только тогда, когда у него **строго больше одного ребенка** в дереве обхода.

### 5. Алгоритм Косарайю и Шарира (Kosaraju-Sharir)

Этот алгоритм применяется для поиска **Сильно Связных Компонент (ССК, SCC)** в *ориентированном* графе.
*   **ССК** — это максимальное подмножество вершин орграфа, в котором от любой вершины можно добраться до любой другой вершины из этого подмножества.

**Сложность алгоритма:** $O(V + E)$.

**Суть и шаги алгоритма:**
Алгоритм опирается на тот факт, что если транспонировать (развернуть) все ребра графа, то сильно связные компоненты останутся теми же, но связи *между* компонентами изменят направление.

1.  **Шаг 1: Прямой DFS и сортировка по времени выхода.**
    Запускаем DFS на исходном графе $G$. Как только DFS завершает обработку вершины (то есть выходит из неё, проверив всех соседей), мы добавляем эту вершину в список (или стек) $L$. По сути, мы получаем массив вершин, отсортированный по времени выхода (`tout`).
2.  **Шаг 2: Транспонирование графа.**
    Строим граф $G^R$, в котором все ребра исходного графа $G$ заменены на противоположные (из $u \to v$ делаем $v \to u$).
3.  **Шаг 3: Обратный DFS по списку $L$.**
    Берём вершины из списка $L$ в **обратном** порядке (начиная с вершины с максимальным `tout`, которая будет на вершине стека). 
    Если вершина еще не посещена, запускаем из неё новый DFS уже по транспонированному графу $G^R$. 
    Всё множество вершин, которое обойдет этот DFS, образует ровно одну Сильно Связную Компоненту (ССК). Записываем её, возвращаемся к списку $L$ и ищем следующую непосещенную вершину.

**Почему это работает?**
Вершина с максимальным `tout` после первого шага гарантированно находится в компоненте, из которой "никуда нельзя уйти" (компонента-исток) в графе $G$, либо ведет в уже полностью обработанные компоненты. В транспонированном графе $G^R$ она становится "стоком" — из неё нельзя выйти за пределы её собственной компоненты связности. Поэтому второй DFS изолирует ровно одну компоненту за запуск.

### 6. Метрические характеристики графа

Метрические характеристики изучают "расстояния" в графах.
*Расстояние* $d(u, v)$ между вершинами $u$ и $v$ — это длина кратчайшего пути между ними (количество ребер). Если пути нет, расстояние считается равным бесконечности.

Для вычисления метрических характеристик для каждой вершины определяется её **эксцентриситет $e(v)$** — это расстояние от вершины $v$ до *самой дальней* от неё вершины в графе. $e(v) = \max_{u \in V} d(v, u)$. Простыми словами: "сколько минимум шагов нужно сделать, чтобы из $v$ гарантированно дойти до любой другой вершины".

На основе эксцентриситета строятся следующие характеристики графа $G$:
*   **Радиус $R(G)$:** Минимальный из эксцентриситетов всех вершин графа. $R(G) = \min_{v \in V} e(v)$.
*   **Диаметр $D(G)$:** Максимальный из эксцентриситетов. Это наибольшее расстояние между какими-либо двумя вершинами графа. $D(G) = \max_{v \in V} e(v)$.
*   **Центр графа:** Множество вершин, чей эксцентриситет равен радиусу графа ($e(v) = R(G)$). У дерева центром может быть либо одна вершина, либо две смежные.
*   **Периферия графа:** Множество вершин, чей эксцентриситет равен диаметру ($e(v) = D(G)$).
*   *Дополнительные термины:* **Обхват** — длина кратчайшего цикла в графе; **Окружность** — длина самого длинного простого цикла.

### 7. Эйлеровы графы. Алгоритмы на основе DFS и Флёри

*   **Эйлеров путь** — путь в графе, проходящий по каждому ребру ровно один раз.
*   **Эйлеров цикл** — замкнутый эйлеров путь.
*   **Эйлеров граф** — граф, содержащий эйлеров цикл.

**Теорема Эйлера (условия существования):**
1.  *Для эйлерова цикла:* Неориентированный связный граф имеет эйлеров цикл тогда и только тогда, когда степени **всех** его вершин четные.
2.  *Для эйлерова пути:* Связный граф имеет эйлеров путь тогда и только тогда, когда ровно **две** вершины имеют нечетную степень (они будут началом и концом пути), а остальные — четную.

**Алгоритм Флёри (исторический, но медленный):**
Суть: строим путь, шаг за шагом удаляя пройденные ребра. Главное правило — **"не жги мосты, пока есть другой выход"**.
1.  Начинаем в вершине с нечетной степенью (если есть) или в любой (для цикла).
2.  На каждом шаге из текущей вершины выбираем любое инцидентное ребро. Но если одно из ребер является мостом (его удаление распадёт оставшийся граф на части), выбираем его *только в том случае*, если других ребер нет.
3.  Переходим по ребру, удаляем его из графа.
*Сложность:* $O(E^2)$, так как на каждом шаге нужно проверять, не является ли ребро мостом (это требует времени).

**Алгоритм Иерахолца (через DFS — эффективный):**
Суть: идем вглубь (как в DFS), пока не застрянем. Застрять мы можем только в стартовой вершине (если степени всех вершин четные).
1.  Начинаем из подходящей вершины $v$. Идем по любому неиспользованному ребру в вершину $u$, удаляем это ребро.
2.  Вызываем функцию рекурсивно от $u$.
3.  Если из вершины больше нет исходящих неиспользованных ребер, **кладем эту вершину в стек (или добавляем в конец массива ответа)** и возвращаемся (backtrack).
4.  После обхода разворачиваем массив ответа.
*Сложность:* $O(V + E)$ (фактически $O(E)$ для связного графа).

### 8. Графы де Брюина и их применения

**Граф де Брюина** $B(k, n)$ — это специальный ориентированный граф, который показывает все возможные перекрытия строк длины $n$, состоящих из алфавита размера $k$.

*   **Вершины:** Все возможные слова длины $n-1$ (их количество $k^{n-1}$).
*   **Ребра:** Все возможные слова длины $n$ (их количество $k^n$).
*   **Связь:** Дуга идет от вершины $A$ к вершине $B$, если суффикс длины $n-2$ слова $A$ совпадает с префиксом слова $B$. Например, при алфавите $\{0,1\}$ из вершины `01` идут дуги в `10` (по ребру `010`) и в `11` (по ребру `011`).

**Свойства:**
В графе де Брюина полустепень захода любой вершины равна полустепени исхода и равна $k$. Значит, по теореме Эйлера, в нём **всегда существует эйлеров цикл**.
Эйлеров цикл в этом графе формирует *последовательность де Брюина* — кратчайшую строку (длины $k^n$), которая содержит в себе *все* возможные подстроки длины $n$ по одному разу.

**Применения:**
1.  **Биоинформатика (сборка генома):** Секвенаторы ДНК читают геном в виде миллионов коротких фрагментов (ридов). Чтобы собрать из них исходный геном, строят граф де Брюина, где фрагменты выступают в роли ребер. Эйлеров путь в таком графе и будет являться собранной последовательностью ДНК.
2.  **Криптография и радиотехника:** Генерация псевдослучайных последовательностей (через регистры сдвига с линейной обратной связью).

### 9. Гамильтоновы графы. Теорема Оре, турниры

*   **Гамильтонов путь** — простой путь, который проходит через каждую вершину графа **ровно один раз** (в отличие от эйлерова, где важны ребра).
*   **Гамильтонов цикл** — замкнутый гамильтонов путь.
*   **Гамильтонов граф** — граф, содержащий гамильтонов цикл.

*Важно:* В отличие от эйлерова цикла, задача поиска гамильтонова цикла является **NP-полной**. Простых необходимых и достаточных условий его существования (как четность степеней) для произвольного графа не существует.

Существуют только *достаточные* условия (если условие выполнено, граф точно гамильтонов, но если нет — он всё равно может им быть). Одно из них — **Теорема Оре (1960 г.)**:
> Если в неориентированном простом графе $G$ с $n \ge 3$ вершинами для *любых двух несмежных* вершин $u$ и $v$ выполняется неравенство: $\deg(u) + \deg(v) \ge n$, то граф является гамильтоновым.

**Турниры:**
*Турнир* — это ориентированный граф, в котором каждые две вершины соединены ровно одной дугой (возникает, если в полном неориентированном графе расставить направления ребрам; интерпретируется как результат турнира, где "каждый сыграл с каждым" без ничьих).
*   **Теорема Редеи:** В *любом* турнире всегда есть направленный гамильтонов путь (доказывается по математической индукции).
*   **Теорема Камьона:** Сильно связный турнир всегда содержит направленный гамильтонов цикл.

### 10. Деревья, теорема Кэли, код Прюфера, корневые деревья

**Дерево** — это связный неориентированный граф без циклов.
Свойства: $|E| = |V| - 1$; между любыми двумя вершинами существует ровно один простой путь.
**Корневое дерево** — дерево, в котором одна вершина выделена и названа "корнем". Это задает дереву иерархическую структуру (появляются понятия "предок", "потомок", "высота дерева"). Удобно для структур данных (например, бинарные деревья поиска).

**Теорема Кэли:**
Количество различных остовных деревьев в полном графе $K_n$ (на $n$ помеченных вершинах) равно $n^{n-2}$.
*(Помеченные вершины означают, что деревья $A-B-C$ и $B-A-C$ считаются разными, так как связи между конкретными узлами разные).*

**Код Прюфера (Prüfer sequence):**
Это элегантный способ доказать теорему Кэли и представить любое дерево на $n$ вершинах в виде последовательности длины $n-2$ из чисел от 1 до $n$. Между множеством деревьев и множеством таких кодов существует биекция (взаимно однозначное соответствие).

**Как построить код Прюфера из дерева:**
1.  Находим все листья дерева (вершины степени 1).
2.  Выбираем лист с **минимальным номером**.
3.  Записываем в код номер вершины, с которой этот лист соединен (соседа).
4.  Удаляем этот минимальный лист из дерева вместе с ребром.
5.  Повторяем шаги 1–4, пока в дереве не останется ровно 2 вершины. Алгоритм завершается, у нас в руках последовательность длины $n-2$.

**Свойства кода:**
*   Вершина $v$ встречается в коде Прюфера ровно $\deg(v) - 1$ раз.
*   По коду можно однозначно восстановить исходное дерево (на этом основан алгоритм восстановления, где мы используем знание, что вершина степени 1 — это та, которой *нет* в текущем остатке кода).

### 11. Каркасы. Главные циклы и разрезы. Алгоритм Каргера

**Каркас (остовное дерево, spanning tree)** графа $G$ — это подграф, который содержит все вершины графа $G$ и при этом является деревом (связан и не имеет циклов). В связном графе с $V$ вершинами любой каркас всегда содержит ровно $V - 1$ ребер. Ребра графа, не вошедшие в каркас, называются *хордами*.

**Главные (фундаментальные) циклы:**
Если к каркасу добавить ровно одну любую хорду, то в образовавшемся графе появится **ровно один цикл**. Этот цикл называется фундаментальным. 
Множество всех таких циклов (соответствующих каждой хорде) образует *фундаментальный базис циклов*. Любой другой цикл в графе можно получить через операцию симметрической разности (XOR) ребер фундаментальных циклов.
*Количество главных циклов* (цикломатическое число) равно числу хорд: $\nu(G) = |E| - |V| + 1$.

**Главные (фундаментальные) разрезы:**
Разрез — это множество ребер, удаление которых делит граф на две несвязные части.
Если в каркасе удалить ровно одно ребро, каркас распадется на два дерева. Если теперь собрать все ребра исходного графа $G$, которые соединяют эти два дерева, то мы получим фундаментальный разрез. У графа ровно $V - 1$ главных разрезов (по числу ребер в каркасе).

**Алгоритм Каргера (Karger's algorithm):**
Это рандомизированный (вероятностный) алгоритм для нахождения **глобального минимального разреза** в связном неориентированном мультиграфе. Минимальный разрез — это минимальное количество ребер, удаление которых делает граф несвязным.
*   **Суть (стягивание ребра):** Алгоритм на каждом шаге случайным образом (равновероятно) выбирает ребро и "стягивает" его: две вершины, инцидентные ребру, сливаются в одну. При этом удаляются петли (ребра, ведущие из вершины в саму себя), но сохраняются кратные ребра между новой и остальными вершинами.
*   **Шаги:** Процесс стягивания повторяется, пока в графе не останутся ровно 2 вершины. Ребра, соединяющие эти две вершины, и есть найденный разрез.
*   **Вероятность успеха:** Вероятность того, что один запуск алгоритма найдет *истинный* минимальный разрез, равна $\approx \frac{2}{V(V-1)}$. Поскольку вероятность мала, алгоритм запускают $O(V^2 \log V)$ раз и выбирают минимальный результат из всех запусков.

### 12. Двудольные графы. Критерий и алгоритм определения двудольности

**Двудольный граф** — это граф, множество вершин которого можно разбить на две непересекающиеся части (доли) $V_1$ и $V_2$ так, что **ни одно ребро не соединяет вершины внутри одной доли** (каждое ребро соединяет вершину из $V_1$ с вершиной из $V_2$).

**Критерий (Теорема Кёнига):**
> Граф является двудольным тогда и только тогда, когда он не содержит циклов нечетной длины.

**Алгоритм определения двудольности (на основе раскраски за $O(V + E)$):**
Используется обход в ширину (BFS) или в глубину (DFS). Суть состоит в попытке раскрасить вершины графа в два цвета (например, 1 и 2).
1.  Берем любую нераскрашенную вершину и красим её в цвет 1.
2.  Обходим всех её соседей и красим их в цвет 2.
3.  Обходим соседей соседей и красим их в цвет 1 (чередуем цвета).
4.  Если в процессе обхода мы пытаемся перекрасить соседа в цвет, отличный от того, который он уже имеет (или, другими словами, находим ребро между вершинами одного цвета), алгоритм останавливается: **граф не двудольный**.
5.  Если граф содержит несколько компонент связности, процедуру нужно запустить для каждой нераскрашенной компоненты. Если удалось раскрасить все вершины — граф двудольный, а цвета вершин обозначают их принадлежность к долям $V_1$ и $V_2$.

### 13. Паросочетания, определения, построение наибольшего паросочетания

**Определения:**
*   **Паросочетание (Matching) $M$** — это набор попарно несмежных ребер графа (то есть никакие два ребра из $M$ не имеют общей вершины).
*   **Свободная вершина** — вершина, не инцидентная ни одному ребру из паросочетания.
*   **Максимальное паросочетание (Maximal)** — паросочетание, в которое нельзя добавить ни одно ребро так, чтобы оно осталось паросочетанием.
*   **Наибольшее паросочетание (Maximum)** — паросочетание с *максимально возможным количеством ребер* в данном графе.
*   **Чередующаяся цепь** — путь, в котором ребра поочередно принадлежат и не принадлежат $M$.
*   **Увеличивающая цепь (Augmenting path)** — чередующаяся цепь, которая **начинается и заканчивается в свободных вершинах**.

**Теорема Бержа:** Паросочетание $M$ является наибольшим тогда и только тогда, когда в графе не существует увеличивающей цепи по отношению к $M$.

**Построение наибольшего паросочетания (Алгоритм Куна для двудольных графов):**
Если в графе есть увеличивающая цепь, то, "перевернув" её (исключив из $M$ ребра, которые там были, и добавив те, которых не было), мы увеличим размер паросочетания ровно на 1.
*Алгоритм (Сложность $O(V \cdot E)$):*
1.  Изначально паросочетание пустое.
2.  Перебираем все вершины левой доли $V_1$. Для каждой вершины пытаемся найти увеличивающую цепь (с помощью DFS или BFS).
3.  Если цепь найдена, делаем "переворот" (чередование) статусов ребер на этом пути и размер паросочетания увеличивается.
4.  Если ни из одной свободной вершины левой доли больше нельзя найти увеличивающую цепь, паросочетание является наибольшим.
*(Для произвольных, не двудольных графов используется гораздо более сложный **алгоритм сжатия соцветий Эдмондса**).*

### 14. Клики и независимые множества

Эти понятия описывают "плотность" и "разреженность" связей в графах. Поиск наибольших таких структур — **NP-полная задача**.

*   **Клика (Clique)** — это подмножество вершин графа, в котором *каждые две вершины соединены ребром* (то есть клика индуцирует полный подграф).
    *   *Кликовое число $\omega(G)$* — размер (число вершин) наибольшей клики в графе.
*   **Независимое множество (Independent set / Антиклика)** — это подмножество вершин графа, в котором *никакие две вершины не соединены ребром*.
    *   *Число независимости $\alpha(G)$* — размер наибольшего независимого множества.
*   **Связь:** Множество вершин является кликой в графе $G$ тогда и только тогда, когда оно является независимым множеством в *дополнении графа* $\overline{G}$ (граф, где ребра есть там, где их не было в $G$, и наоборот).

**Вершинное покрытие (Vertex Cover):**
Это множество вершин $C$, такое, что каждое ребро графа инцидентно хотя бы одной вершине из $C$.
*   *Связь (Теорема Галлаи):* Для любого графа $G$ без изолированных вершин с $V$ вершинами выполняется равенство: $\alpha(G) + \tau(G) = |V|$, где $\tau(G)$ — размер наименьшего вершинного покрытия. То есть, если $I$ — наибольшее независимое множество, то оставшиеся вершины $V \setminus I$ образуют наименьшее вершинное покрытие.

### 15. Изоморфизм графов. Инварианты

**Изоморфизм:**
Два графа $G_1 = (V_1, E_1)$ и $G_2 = (V_2, E_2)$ называются изоморфными, если между их вершинами существует биекция (взаимно однозначное соответствие) $f: V_1 \to V_2$ такая, что любые две вершины $u, v$ соединены ребром в $G_1$ тогда и только тогда, когда вершины $f(u), f(v)$ соединены ребром в $G_2$.
*Простыми словами:* Это один и тот же граф, просто с переименованными вершинами и "нарисованный" по-разному на плоскости. Задача изоморфизма графов известна своей неопределенной сложностью (класс GI — не доказано, что она P, и не доказано, что NP-полная).

**Инварианты графа:**
Инвариант — это некоторое свойство или числовая характеристика графа, которая **не меняется** при изоморфизме. 
Если у двух графов какой-либо инвариант *отличается*, то они **точно не изоморфны**. Но если все известные инварианты *совпадают*, это еще не гарантирует изоморфизм (графы все равно могут быть разными).

*Примеры важнейших инвариантов:*
1.  Количество вершин $|V|$ и количество ребер $|E|$.
2.  **Степенная последовательность:** Упорядоченный по убыванию список степеней всех вершин.
3.  Количество компонент связности.
4.  Наличие Эйлерова или Гамильтонова цикла.
5.  Метрические характеристики (радиус, диаметр).
6.  Кликовое число $\omega(G)$ и число независимости $\alpha(G)$.
7.  **Спектр графа:** Набор собственных значений матрицы смежности. (Графы с одинаковым спектром называются коспектральными, и они могут быть не изоморфны).

### 16. Планарность. Грани графа, теорема Эйлера и ее применения

**Планарный граф** — это граф, который можно изобразить на плоскости так, чтобы его ребра пересекались **только в вершинах**. Конкретное изображение планарного графа на плоскости без пересечения ребер называется *плоским графом*.

**Грани графа:**
Любой плоский граф разбивает плоскость на непрерывные области, которые называются гранями.
*   Каждая грань ограничена циклом из ребер.
*   Всегда существует ровно одна **внешняя (бесконечная) грань**, которая окружает весь граф.

**Теорема (Формула) Эйлера:**
Для любого связного плоского графа справедливо соотношение:
$$V - E + F = 2$$
где $V$ — количество вершин, $E$ — количество ребер, $F$ — количество граней (включая внешнюю).

**Важные следствия и применения теоремы Эйлера:**
Формула Эйлера используется для доказательства того, что некоторые графы **нельзя** нарисовать на плоскости.
1.  **Оценка числа ребер:** В любом простом связном планарном графе с $V \ge 3$ выполняется: **$E \le 3V - 6$**. 
    *   *Применение:* Полный граф $K_5$ (5 вершин, 10 ребер) не планарен, так как $10 \not\le 3 \cdot 5 - 6$ (т.е. $10 \not\le 9$).
2.  **Оценка для графов без треугольников:** Если в простом планарном графе нет циклов длины 3 (например, двудольные графы), то для $V \ge 3$ выполняется: **$E \le 2V - 4$**.
    *   *Применение:* Полный двудольный граф $K_{3,3}$ (домики и колодцы: 6 вершин, 9 ребер) не планарен, так как $9 \not\le 2 \cdot 6 - 4$ (т.е. $9 \not\le 8$).
3.  В любом простом планарном графе всегда найдется **хотя бы одна вершина степени $\le 5$**.

### 17. Гомеоморфизм графов. Критерий Понтрягина-Куратовского

Чтобы алгоритмически или теоретически понять, планарен ли произвольный граф, используют критерий Куратовского.

**Подразделение ребра и гомеоморфизм:**
*   *Подразделение ребра (subdivision)* — это операция, при которой одно ребро $(u, v)$ удаляется, и вместо него добавляется новая вершина $w$ и два ребра $(u, w)$ и $(w, v)$. По сути, мы ставим точку на ребре.
*   Два графа называются **гомеоморфными**, если они могут быть получены из одного и того же графа путем многократного подразделения ребер. (Простыми словами: графы отличаются только вершинами степени 2, "навешанными" на ребра).

**Теорема Понтрягина-Куратовского:**
> Граф является планарным тогда и только тогда, когда он **не содержит** подграфа, гомеоморфного графу $K_5$ (полный на 5 вершинах) или графу $K_{3,3}$ (полный двудольный 3 на 3).

*Суть:* Графы $K_5$ и $K_{3,3}$ — это "минимальные" непланарные графы. Если внутри вашего графа спрятана структура хотя бы одного из них (даже если ребра растянуты дополнительными вершинами), граф на плоскости не разложить.

### 18. Вершинная раскраска графов, алгоритм последовательной раскраски

**Правильная вершинная раскраска** — это назначение каждому узлу графа определенного цвета так, чтобы любые две смежные вершины (соединенные ребром) имели **разные цвета**.
*   **Хроматическое число $\chi(G)$** — минимальное количество цветов, необходимое для правильной раскраски графа.

*(Например, $\chi(K_n) = n$, $\chi($двудольного графа$) = 2$, $\chi($дерева$) = 2$, по теореме о 4-х красках для любого планарного графа $\chi(G) \le 4$).*

**Алгоритм последовательной раскраски (Жадный алгоритм):**
Это простой и быстрый алгоритм, который раскрашивает граф, но не всегда гарантирует минимальное число цветов.
1.  Упорядочиваем все вершины графа в некоторую последовательность: $v_1, v_2, \dots, v_n$.
2.  Берем первую вершину $v_1$ и красим её в цвет №1.
3.  Для каждой следующей вершины $v_i$ мы просматриваем уже раскрашенных её соседей.
4.  Назначаем $v_i$ **наименьший доступный номер цвета** (1, 2, 3...), который еще не занят ни одним из её смежных соседей.
*   **Сложность:** $O(V + E)$.
*   **Проблема:** Качество результата сильно зависит от порядка вершин. В худшем случае жадный алгоритм может использовать $\Delta(G) + 1$ цветов (где $\Delta$ — максимальная степень вершины). 
*   *Улучшение (Алгоритм Уэлша-Пауэлла):* Перед запуском жадного алгоритма сортируем вершины по *убыванию их степеней*. Это сильно уменьшает шанс использовать лишние цвета.

### 19. Хроматические многочлены: методы редукции графов

**Хроматический многочлен $P(G, \lambda)$** — это функция, которая показывает, сколькими способами можно правильно раскрасить граф $G$, используя **не более $\lambda$ цветов**.
Для любого графа эта функция всегда является полиномом (многочленом) от переменной $\lambda$.

Построить многочлен для сложных графов тяжело, поэтому используют формулы редукции (сведения задачи к более простым графам).
Пусть $e = (u, v)$ — ребро. Введем операции:
*   $G - e$ — граф $G$ с удаленным ребром $e$.
*   $G / e$ — граф $G$, в котором ребро $e$ стянуто (вершины $u$ и $v$ слиты в одну, кратные ребра удалены).

**Метод 1: Редукция к нуль-графам (Формула Зыкова / Удаление-Стягивание)**
Используется формула:
$$P(G, \lambda) = P(G - e, \lambda) - P(G / e, \lambda)$$
Мы берем ребро графа и "разбиваем" расчет на два: в одном ребра нет (ребер стало меньше), в другом вершины слились (вершин стало меньше). Повторяя это рекурсивно, мы сведем граф к набору **пустых графов (нуль-графов)**, то есть графов без ребер (одни изолированные вершины). Для них многочлен известен (см. вопрос 20).

**Метод 2: Редукция к полным графам (Добавление-Стягивание)**
Если $u$ и $v$ **не соединены** ребром, обозначим $G+e$ как граф, где мы добавили ребро между ними. Формула принимает вид:
$$P(G, \lambda) = P(G + e, \lambda) + P(G / e, \lambda)$$
Здесь мы, наоборот, добавляем ребра, пока граф не распадется на набор **полных графов** ($K_n$). Для полных графов многочлен тоже легко пишется.

### 20. Простейшие способы вычисления хроматического многочлена

Вместо долгих редукций для некоторых стандартных классов графов многочлены можно выписать сразу (комбинаторно).

1.  **Нуль-граф / Пустой граф $E_n$** (n вершин, 0 ребер):
    Каждую вершину можно красить в любой из $\lambda$ цветов независимо.
    $$P(E_n, \lambda) = \lambda^n$$
2.  **Полный граф $K_n$** (все вершины соединены):
    Первую красим в любой из $\lambda$ цветов. Вторую — в $\lambda-1$ (т.к. нельзя как первую), третью — в $\lambda-2$ и т.д.
    $$P(K_n, \lambda) = \lambda(\lambda - 1)(\lambda - 2) \dots (\lambda - n + 1)$$
3.  **Дерево $T_n$** (на $n$ вершинах):
    Корневую вершину красим в $\lambda$ цветов. Любую следующую (спускаясь по ребрам) мы обязаны покрасить в цвет, отличный от её единственного предка, то есть выбор из $\lambda-1$ цветов.
    $$P(T_n, \lambda) = \lambda(\lambda - 1)^{n - 1}$$
4.  **Граф из несвязных компонент:**
    Если граф состоит из независимых кусков (компонент связности) $C_1, C_2, \dots, C_k$, то раскраски независимы (правило умножения):
    $$P(G, \lambda) = P(C_1, \lambda) \cdot P(C_2, \lambda) \dots P(C_k, \lambda)$$
5.  **Склейка по одной вершине (наличие шарнира):**
    Если два графа $G_1$ и $G_2$ имеют ровно одну общую вершину (точку сочленения), то их хроматический многочлен:
    $$P(G, \lambda) = \frac{P(G_1, \lambda) \cdot P(G_2, \lambda)}{\lambda}$$
6.  **Простой цикл $C_n$:** *(часто просят на экзамене)*
    $$P(C_n, \lambda) = (\lambda - 1)^n + (-1)^n(\lambda - 1)$$
